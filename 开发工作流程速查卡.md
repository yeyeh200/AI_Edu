# 开发工作流程速查卡

## 🚀 开发任务启动三步法

### 1️⃣ 询问通用问题（必须执行）
```
✅ 您需要我完成什么任务？
✅ 这个任务要达成的业务目标是什么？
✅ 当前进行到哪一步了？
✅ 我需要先阅读哪些文档来了解完整背景？
```

### 2️⃣ 查阅核心文档（按顺序执行）
```
1️⃣ docs/设计文档_MVP/01-需求与范围/01-MVP需求规格说明书.md
2️⃣ docs/设计文档_MVP/02-架构设计/01-系统架构设计.md
3️⃣ 根据任务类型查阅专项设计文档
   - API开发 → API接口规范
   - UI开发 → UI设计规范
   - 数据库 → 数据模型设计
   - 测试 → 测试策略设计
```

### 3️⃣ 一致性检查（关键控制点）
```
✅ 功能一致性：实现功能是否与需求规格一致？
✅ 架构一致性：实现方案是否符合系统架构？
✅ 接口一致性：API是否与接口规范一致？
✅ UI一致性：界面是否与设计规范一致？
```

## 📋 核心验证检查点

### 里程碑1：核心功能开发完成
```
✅ 数据集成：文件上传→解析→存储→错误处理
✅ AI分析：外部集成→多维分析→结果存储→异常处理
✅ 结果可视化：图表展示→数据导出→响应式适配
✅ 用户认证：预设登录→会话管理→权限控制
```

### 里程碑2：前后端集成
```
✅ API集成：接口调用→错误处理→数据传输→性能
✅ 用户界面：页面渲染→交互流程→用户体验→兼容性
```

### 里程碑3：MVP功能验证
```
✅ 完整流程：登录→上传→分析→展示→导出
✅ 性能验证：响应时间→并发处理→资源使用
```

## ⚠️ AI缺陷预防措施

### 防失忆
```
📝 开发日志：每个步骤完成后记录
📸 状态快照：每个里程碑完成后保存
🔄 定期回顾：定期回顾文档和实现
```

### 防急于完成
```
🎯 目标重述：开发前重述任务目标和价值
⏰ 进度检查：每30分钟检查目标对齐
🏆 质量优先：坚持质量优先于进度
```

### 防经验主义
```
📊 事实依据：每个决策都要有文档或测试依据
📖 文档优先：严格遵循设计文档
👥 用户需求：所有功能回溯到用户需求
```

## 🔄 决策框架

### 基于证据的决策
```
1️⃣ 证据收集：检查现有实现和测试结果
2️⃣ 证据评估：分析与目标的符合度
3️⃣ 差距分析：识别与设计的偏差
4️⃣ 决策制定：基于证据决定后续行动
```

### 当前状态分析
```
📋 文档证据：实现与设计对比、测试报告、代码质量
🧪 测试证据：单元测试覆盖率、集成测试结果、UAT结果
👥 用户证据：用户反馈、使用行为、性能监控
```

## 🚨 紧急情况处理

### 发现与设计文档冲突
```
⛔ 立即停止 → 📝 记录冲突点 → 🔍 分析原因 → ❓ 寻求确认 → 📄 文档更新
```

### 发现理解错误
```
✅ 承认错误 → 🔄 重新确认 → 🔧 方案调整 → 📊 影响评估 → ✅ 修正实施
```

## 💡 快速提醒

### 开发前必读
```
🎯 我明确要完成的任务目标吗？
📚 我查阅了所有必需的设计文档吗？
✅ 我的实现方案与文档一致吗？
```

### 开发中常检
```
⏰ 每30分钟检查：我还在正确的方向上吗？
📋 每个功能完成后：是否与设计文档一致？
🧪 每次提交前：测试是否通过？
```

### 开发后必验
```
🔍 功能完整性：所有必需功能都实现了吗？
📊 性能达标：响应时间和资源使用正常吗？
👥 用户体验：操作流程是否顺畅？
```
