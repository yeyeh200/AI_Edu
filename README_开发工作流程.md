## 🚀 快速开始

### 第一步：了解工作流程
1. 阅读 [开发工作流程速查卡.md](./开发工作流程速查卡.md) 了解基本流程
2. 详细阅读 [开发工作流程规则.md](./开发工作流程规则.md) 理解所有规则

### 第二步：开始开发任务
1. 使用"开发任务启动三步法"开始每个任务
2. 按照"文档依赖验证机制"查阅相关文档
3. 使用"开发实施检查清单"进行过程控制

### 第三步：验证和交付
1. 按照"里程碑验证检查点"进行阶段性验证
2. 使用"基于证据的决策框架"进行质量判断
3. 完成后更新开发日志

## ⚠️ 重要提醒

### AI缺陷预防
- **失忆预防**：严格遵守开发日志制度和状态快照
- **目标偏离预防**：坚持目标重述确认和进度检查
- **经验主义预防**：坚持事实依据验证和文档优先

### 质量控制
- **文档一致性**：每个实现都必须与设计文档一致
- **验证完整性**：每个里程碑都必须通过全面验证
- **证据充分性**：每个决策都必须有充分证据支持

## 🔄 持续改进

### 规则更新
- 在开发过程中发现问题及时更新规则
- 根据实际情况调整检查清单
- 定期回顾和完善工作流程

### 经验积累
- 记录开发过程中的最佳实践
- 总结常见问题和解决方案
- 分享成功经验和失败教训

## 📞 问题处理

### 常见问题
1. **任务目标不明确** → 使用通用问题询问原则
2. **与设计文档冲突** → 立即停止并分析原因
3. **开发方向偏离** → 使用里程碑验证检查点
4. **质量标准不明确** → 参考测试策略设计

### 紧急情况
- 立即停止当前工作
- 查阅相关规则文档
- 使用检查清单进行验证
- 寻求明确的指导
