# 用户故事：督导专家查看评价结果

---

**用户故事编号：** US-SUP-001
**用户故事标题：** 督导专家查看负责范围的教师评价结果
**优先级：** 高
**关联功能需求：** FR-001, FR-004
**预估工作量：** 5人天
**创建日期：** 2025-11-23
**最后更新：** 2025-11-23

---

## 1. 基本信息

### 1.1 用户故事描述
**作为一个** 教学督导专家
**我希望** 查看我所负责范围内教师的教学质量评价结果
**以便于** 进行专业的质量评估、提供指导建议并监督评价过程

### 1.2 用户角色定义
- **主要用户角色：** 教学督导专家（SUP）- 负责教学质量监督和评估的专业人员
- **次要用户角色：** 教学督导组长、教学质量管理部门负责人
- **使用场景：** 督导专家查看所负责院系或专业的教师评价结果

### 1.3 业务价值
- **用户价值：** 为督导专家提供专业的评价工具和数据支撑
- **业务价值：** 确保教学质量评价的专业性和权威性
- **衡量指标：** 督导覆盖率≥95%，评价一致性≥90%

---

## 2. 验收标准 (Acceptance Criteria)

### 2.1 主要验收标准
**Given** 督导专家已登录系统且具有督导权限
**When** 督导专家进入"督导评价"模块并选择负责范围
**Then** 系统应该显示该范围内所有教师的评价结果和详细数据
**And** 支持专业评估工具和督导记录功能

### 2.2 详细验收条件
- [ ] **范围控制：** 只显示督导专家负责范围的教师评价结果
- [ ] **专业评价：** 提供专业的教学评估工具和标准
- [ ] **评价详情：** 显示教师的详细评价数据和计算依据
- [ ] **对比分析：** 支持教师间的横向对比和历史对比
- [ ] **督导记录：** 支持督导专家记录专业意见和建议
- [ ] **评价确认：** 督导专家可以对评价结果进行专业确认
- [ ] **报告生成：** 支持生成督导评估报告

### 2.3 边界条件测试
- **正常流程：** 督导专家查看负责范围内教师的正常评价结果
- **异常处理：** 负责范围无教师或数据不足的处理
- **权限测试：** 超出负责范围的访问控制

---

## 3. 业务规则

### 3.1 功能规则
- **责任范围：** 督导专家只能查看所负责院系或专业的教师数据
- **评价标准：** 使用统一的教学质量评价标准体系
- **专业权限：** 督导专家具有专业评价的权限和责任
- **记录要求：** 督导过程需要记录专业意见和建议
- **时间要求：** 督导评价需要在规定时间内完成

### 3.2 数据规则
- **数据准确性：** 评价结果数据准确无误
- **数据完整性：** 提供完整的评价数据链条
- **数据时效性：** 评价数据及时更新
- **数据安全：** 保护教师的评价数据隐私

### 3.3 操作约束
- **权限控制：** 严格控制督导专家的数据访问范围
- **操作记录：** 所有督导操作需要记录日志
- **评价时间：** 在规定时间内完成督导评价工作
- **专业要求：** 需要具备相应的专业资质

---

## 4. 交互流程

### 4.1 用户操作流程
1. **步骤1：** 督导专家登录系统，进入督导工作台
2. **步骤2：** 选择"督导评价"功能模块
3. **步骤3：** 选择要查看的负责范围（院系/专业）
4. **步骤4：** 系统显示该范围内的教师列表和评价结果
5. **步骤5：** 督导专家查看具体教师的详细评价数据
6. **步骤6：** 记录专业评价意见和建议
7. **步骤7：** 生成督导评估报告

### 4.2 系统响应流程
1. **响应1：** 验证督导专家身份和权限
2. **响应2：** 根据责任范围获取教师评价数据
3. **响应3：** 展示教师评价结果列表和统计信息
4. **响应4：** 提供详细评价数据的查看功能
5. **响应5：** 支持督导记录和意见录入
6. **响应6：** 生成专业的督导评估报告

### 4.3 异常处理流程
- **异常情况1：** 负责范围无教师时显示相应提示
- **异常情况2：** 数据不完整时标注并提供补充建议
- **异常情况3：** 权限异常时显示权限范围说明

## 5. 数据需求

### 5.1 输入数据要求
| 数据项 | 数据类型 | 是否必填 | 格式要求 | 取值范围 | 说明 |
|--------|----------|----------|----------|----------|------|
| 督导工号 | String | 是 | 工号格式 | 学校标准工号 | 督导专家身份标识 |
| 负责范围 | String | 是 | 范围代码 | 院系/专业代码 | 督导负责的院系或专业 |
| 评价类型 | String | 否 | 类型代码 | 定期/专项/随机 | 督导评价的类型 |

### 5.2 输出数据格式
| 数据项 | 数据类型 | 格式 | 说明 |
|--------|----------|------|------|
| 教师列表 | Array | JSON数组 | 负责范围内的教师信息 |
| 评价结果 | Array | JSON数组 | 教师的评价结果详情 |
| 统计信息 | Object | JSON对象 | 范围内的统计汇总数据 |
| 督导记录 | Array | JSON数组 | 督导专家的评价记录 |
| 对比分析 | Object | JSON对象 | 教师间的对比分析数据 |

### 5.3 数据验证规则
- **权限验证：** 确保督导专家只能访问负责范围的数据
- **数据验证：** 验证评价结果的准确性和完整性
- **业务验证：** 验证评价数据符合业务规则
