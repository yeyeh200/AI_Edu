## 业务流程关联关系

### 主要流程依赖关系

```mermaid
graph LR
    A[用户管理流程] --> B[权限管理流程]
    B --> C[评估任务创建和管理流程]
    C --> D[评估执行流程]
    D --> E[报告生成流程]
    A --> F[系统管理流程]
    F -.-> C
    F -.-> D
    F -.-> E
```

### 关键节点说明

1. **用户管理**是所有流程的基础，确保用户身份的正确性和合法性
2. **权限管理**为其他流程提供安全访问控制
3. **评估任务创建**是评价工作的起点，确定评价的范围和标准
4. **评估执行**是核心分析过程，产生评价结果
5. **报告生成**将分析结果转化为可理解的业务洞察
6. **系统管理**为整个系统提供稳定运行保障
