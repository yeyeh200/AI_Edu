## 1. 引言

### 1.1 文档目的

本文档详细描述了AI助评系统MVP（最小可行产品）版本的功能需求和非功能需求，为开发团队提供明确的需求指导，确保产品开发符合预期目标和质量标准。

### 1.2 项目背景

随着教育数字化转型的深入，传统的教学评价方式存在评价维度单一、数据支撑不足、评价周期长等问题。AI助评系统旨在通过人工智能技术，整合多源教学数据，建立科学的教学质量评价体系，为教学改革提供数据支撑。

MVP阶段的核心目标是验证AI助评系统的核心业务价值，通过数据集成→AI分析→结果展示的完整链路，证明系统的可行性。

### 1.3 MVP目标

#### 1.3.1 核心目标
- **业务价值验证**：验证AI驱动的教学质量评价的可行性
- **技术可行性验证**：验证技术架构和技术方案的可行性
- **用户接受度验证**：验证用户对AI助评系统的接受程度

#### 1.3.2 具体目标
- 建立职教云数据集成的完整流程
- 实现基础的AI评价规则引擎
- 提供直观的可视化分析展示
- 建立简化的用户认证和权限体系

### 1.4 术语定义

| 术语 | 说明 |
|------|------|
| MVP | Minimum Viable Product，最小可行产品 |
| AI助评 | 基于人工智能技术的教学评价系统 |
| 职教云 | 职业教育云平台，主要数据源 |
| 规则引擎 | 基于预设规则进行数据分析和评价的组件 |
| 数据集成 | 从多个数据源收集、清洗、整合数据的过程 |
| 可视化展示 | 通过图表、仪表盘等形式展示数据分析结果 |

## 2. 项目概述

### 2.1 项目愿景

构建基于人工智能技术的智能教学评价平台，通过数据驱动的方式，实现对教师教学质量、课堂教学质量的科学评估，为教学改进和管理决策提供客观依据。

### 2.2 MVP范围定义

#### 2.2.1 范围内功能
1. **简化用户认证**
   - 预置账号登录（管理员、教师）
   - 基础会话管理
   - 简单角色区分

2. **数据集成**
   - 职教云系统数据对接
   - 基础数据采集和清洗
   - 数据质量监控

3. **AI分析引擎**
   - 基础规则引擎
   - 评价指标计算
   - 分析结果存储

4. **可视化展示**
   - 主仪表盘页面
   - 核心指标图表
   - 基础报表功能

5. **系统管理**
   - 基础配置管理
   - 系统监控
   - 日志管理

#### 2.2.2 范围外功能
- 复杂的用户管理系统（用户注册、审批流程等）
- 详细的RBAC权限体系
- 第三方OAuth集成
- 高级AI算法模型
- 移动端应用
- 教务系统、竞赛系统集成
- 复杂的数据分析功能

### 2.3 用户角色定义

#### 2.3.1 管理员（Admin）
- **职责**：
  - 系统配置和管理
  - 数据源配置和监控
  - 评价规则配置
  - 查看全局数据分析结果

- **权限**：
  - 访问系统所有功能
  - 配置数据采集参数
  - 管理评价规则
  - 查看所有数据分析报告

#### 2.3.2 教师（Teacher）
- **职责**：
  - 查看个人教学评价结果
  - 获取教学改进建议
  - 查看班级整体分析数据

- **权限**：
  - 查看个人评价报告
  - 查看所授班级的数据
  - 导出个人分析报表

### 2.4 使用场景分析

#### 2.4.1 管理员使用场景

**场景1：数据源配置**
```
作为管理员，我需要配置职教云数据源连接，以便系统能够自动采集教学数据。
前置条件：管理员已登录
操作步骤：
1. 进入系统配置页面
2. 配置职教云API连接参数
3. 测试连接是否成功
4. 保存配置
预期结果：数据源配置成功，系统可以正常获取职教云数据
```

**场景2：评价规则配置**
```
作为管理员，我需要配置教学评价规则，以便系统能够按照设定的标准进行评价。
前置条件：管理员已登录，数据源配置完成
操作步骤：
1. 进入评价规则配置页面
2. 设置评价指标和权重
3. 配置评价规则逻辑
4. 保存并启用规则
预期结果：评价规则配置成功，系统按新规则进行评价分析
```

**场景3：查看分析结果**
```
作为管理员，我需要查看全校教学质量分析结果，以便了解整体教学状况。
前置条件：管理员已登录，数据分析完成
操作步骤：
1. 进入仪表盘页面
2. 查看各项教学指标
3. 查看详细分析报告
4. 导出分析报表
预期结果：能够看到完整的教学质量分析数据和图表
```

#### 2.4.2 教师使用场景

**场景1：登录系统**
```
作为教师，我需要登录AI助评系统，以便查看我的教学评价结果。
前置条件：教师账号已预置
操作步骤：
1. 打开系统登录页面
2. 输入用户名和密码
3. 点击登录按钮
预期结果：成功登录系统，进入教师仪表盘
```

**场景2：查看个人评价**
```
作为教师，我需要查看我的教学质量评价报告，以便了解教学效果。
前置条件：教师已登录，评价分析完成
操作步骤：
1. 进入个人评价页面
2. 查看评价指标得分
3. 查看详细分析结果
4. 查看改进建议
预期结果：能够看到完整的教学评价报告和改进建议
```

**场景3：查看班级数据**
```
作为教师，我需要查看我授课班级的整体数据，以便了解班级学习情况。
前置条件：教师已登录，班级数据分析完成
操作步骤：
1. 进入班级数据页面
2. 选择要查看的班级
3. 查看班级各项指标
4. 查看学生参与度数据
预期结果：能够看到班级的详细学习数据和参与度分析
```

## 3. 功能需求

### 3.1 核心业务流程需求

#### 3.1.1 数据采集流程

**需求描述**：系统能够自动从职教云平台采集教学相关数据，包括课程信息、学生信息、学习活动数据等。

**功能要求**：
- 支持职教云API对接
- 支持增量数据采集
- 支持数据质量检查
- 支持采集状态监控
- 支持采集日志记录

**输入**：
- 职教云API配置参数
- 数据采集时间范围
- 数据类型过滤条件

**处理**：
- 调用职教云API获取数据
- 验证数据格式和完整性
- 清洗和转换数据
- 存储到本地数据库

**输出**：
- 采集成功/失败状态
- 采集数据记录数
- 数据质量报告
- 采集执行日志

**验收标准**：
- 支持从职教云获取至少10种类型的数据
- 数据采集完整率 ≥ 95%
- 支持增量数据更新
- 提供采集状态实时监控

#### 3.1.2 AI分析流程

**需求描述**：系统基于采集的教学数据，运用规则引擎进行教学质量评价分析。

**功能要求**：
- 支持可配置的评价规则
- 支持多维度指标计算
- 支持分析结果存储
- 支持分析任务调度
- 支持分析过程监控

**输入**：
- 教学基础数据
- 评价规则配置
- 分析参数设置

**处理**：
- 加载评价规则
- 计算各项评价指标
- 生成评价结果
- 保存分析结果

**输出**：
- 教学质量评分
- 评价指标明细
- 改进建议
- 分析报告

**验收标准**：
- 支持5个以上核心评价指标
- 评价结果准确性 ≥ 90%
- 支持批量分析处理
- 提供分析结果查询接口

#### 3.1.3 结果展示流程

**需求描述**：系统能够以图表和报表的形式直观展示教学质量评价结果。

**功能要求**：
- 支持仪表盘展示
- 支持多种图表类型
- 支持数据筛选查询
- 支持报表导出
- 支持实时数据刷新

**输入**：
- 分析结果数据
- 用户查询条件
- 展示参数配置

**处理**：
- 查询分析结果数据
- 生成图表数据
- 渲染可视化组件
- 提供数据导出

**输出**：
- 可视化图表
- 数据报表
- 导出文件

**验收标准**：
- 提供3种以上图表类型
- 支持实时数据刷新
- 页面加载时间 ≤ 3秒
- 支持PDF和Excel导出

### 3.2 简化用户认证需求

#### 3.2.1 用户登录

**需求描述**：系统提供基础的用户登录功能，支持预置账号的登录验证。

**功能要求**：
- 支持用户名密码登录
- 支持登录状态保持
- 支持登录失败处理
- 支持自动登出功能

**输入**：
- 用户名
- 密码

**处理**：
- 验证用户名和密码
- 创建用户会话
- 设置用户权限
- 跳转到主页面

**输出**：
- 登录成功/失败状态
- 用户权限信息
- 会话令牌

**验收标准**：
- 支持管理员和教师两种角色
- 登录响应时间 ≤ 2秒
- 会话有效期24小时
- 支持"记住我"功能

#### 3.2.2 权限控制

**需求描述**：系统提供基于角色的简单权限控制，确保用户只能访问授权的功能。

**功能要求**：
- 支持角色权限验证
- 支持页面访问控制
- 支持API接口权限检查
- 支持权限失效处理

**输入**：
- 用户会话信息
- 访问资源标识

**处理**：
- 检查用户会话有效性
- 验证用户角色权限
- 判断资源访问权限
- 记录权限检查日志

**输出**：
- 访问允许/拒绝结果
- 权限错误信息

**验收标准**：
- 支持页面级权限控制
- 支持API接口权限控制
- 权限检查响应时间 ≤ 100ms
- 记录完整的权限检查日志

### 3.3 数据集成需求

- 提供API调用日志

**需求描述**：系统能够对采集的原始数据进行清洗、转换和标准化处理。

**功能要求**：

- 支持数据去重处理
- 支持数据验证检查

**输入**：
- 原始数据JSON
- 数据处理规则

**处理**：
- 解析原始数据
- 执行数据转换
- 验证数据完整性

**输出**：
- 清洗后数据
- 数据质量报告
- 异常数据记录

**验收标准**：
- 数据清洗准确率 ≥ 99%
- 支持10种以上数据转换规则
- 提供详细的数据质量报告
- 支持异常数据人工审核

### 3.4 AI分析引擎需求

**输出**：
- 指标计算结果

- 指标历史对比

**验收标准**：
- 支持10个以上核心指标
- 指标计算准确率 ≥ 90%
- 指标计算时间 ≤ 30秒
- 支持指标结果导出
