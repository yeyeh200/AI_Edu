# EDU_COURSE_PRACTICE 理论课程/实践环节表

## 基本信息

- **表名**：`EDU_COURSE_PRACTICE`（原表名：`V_BRW_CoursePractice`）
- **中文名称**：理论课程/实践环节
- **用途**：存储课程和实践环节的详细信息，包括课程属性、学时分配、负责人等，用于教学管理
- **字段数量**：19个
- **数据类别**：教务系统 - 教学管理模块

## 字段详细说明

### 主键字段
- **user_kcdm**：课程代码，唯一标识符，主键

### 基本课程信息字段
- **zwmc**：课程中文名称
- **ywmc**：课程英文名称
- **kc_flag**：课程环节标志
  - 0：理论课程
  - 1：实践环节

### 承担单位字段
- **cddw_mc**：承担单位名称
- **cddw_userdm**：承担单位代码

### 学分和学时字段
- **xf**：学分
- **jsxs**：讲授学时
- **syxs**：实验学时
- **sjxs**：上机学时
- **qtxs**：其他学时

### 实践环节特有字段
- **hjlb**：环节类别
- **zs**：周数，仅当 kc_flag='1' 时有效

### 理论课程特有字段
- **KCLB3**：课程类别三，仅当 kc_flag='0' 时有效
- **KCLB4**：课程类别四，仅当 kc_flag='0' 时有效

### 管理字段
- **SFKYKC**：是否可用（1：可用，0：不可用）
- **Kcfzr**：课程负责人工号
- **Kcfzr_XM**：课程负责人姓名

## 课程类型说明

### 理论课程（kc_flag = '0'）
- **定义**：以理论教学为主的课程
- **特点**：主要以课堂讲授、讨论等形式进行
- **学时构成**：以讲授学时为主，可能包含少量实验学时
- **示例**：高等数学、大学英语、思想政治理论课

### 实践环节（kc_flag = '1'）
- **定义**：以实践操作为主的课程
- **特点**：主要以实验、实习、实训等形式进行
- **学时构成**：以实践学时为主，可能包含少量理论学时
- **示例**：金工实习、毕业设计、课程设计

## 学时分配说明

### 总学时构成
```
总学时 = 讲授学时 + 实验学时 + 上机学时 + 其他学时
```

### 不同类型课程的学时侧重点
- **理论课程**：讲授学时占总学时的70%以上
- **实验课程**：实验学时占总学时的50%以上
- **实习课程**：实践学时占总学时的80%以上
- **课程设计**：实践学时占总学时的60%以上

## 课程类别说明

### 课程类别三（KCLB3）
- **必修课程**：必须修读的课程
- **选修课程**：可以选择修读的课程
- **专业核心课**：专业核心要求的课程
- **通识课**：跨学科的通用课程

### 课程类别四（KCLB4）
- **公共基础课**：全校性基础课程
- **学科基础课**：学科内基础理论课程
- **专业基础课**：专业基础理论课程
- **专业核心课**：专业核心理论课程

## 环节类别说明

### 常见环节类别
- **金工实习**：金工实习环节
- **生产实习**：企业生产实习环节
- **毕业设计**：毕业设计（论文）环节
- **课程设计**：课程设计环节
- **认识实习**：专业认识实习
- **社会实践**：社会实践活动
- **实训项目**：实训项目课程

## 数据约束

1. **主键约束**：`user_kcdm` 字段必须唯一且非空
2. **检查约束**：`kc_flag` 只能是 0 或 1
3. **检查约束**：`SFKYKC` 只能是 0 或 1
4. **数值约束**：所有学时字段和学分字段应为非负数
5. **条件约束**：当 `kc_flag='0'` 时，`zs` 字段通常为空或0
6. **条件约束**：当 `kc_flag='1'` 时，`KCLB3` 和 `KCLB4` 字段通常为空

## 索引建议

- **主键索引**：`user_kcdm` - 主键自动创建唯一索引
- **课程类型索引**：`kc_flag` - 用于区分理论和实践课程
- **类别索引**：`KCLB3`, `KCLB4` - 用于按课程类别筛选
- **单位索引**：`cddw_userdm` - 用于按承担单位查询
- **负责人索引**：`Kcfzr` - 用于按负责人查询
- **状态索引**：`SFKYKC` - 用于过滤有效课程

## 数据关联关系

### 主要外键关联
- **院系信息表**：通过 `cddw_userdm` 关联院系信息
- **教师信息表**：通过 `Kcfzr` 关联课程负责人信息

### 业务关联
- **学期开课计划表**：根据课程信息制定开课计划
- **课程安排表**：根据课程信息安排具体授课
- **教材信息表**：根据课程信息确定使用教材
- **考试安排表**：根据课程信息安排考试

## 使用示例

### SQL查询示例

```sql
-- 查询所有理论课程
SELECT user_kcdm, zwmc, xf, jsxs, syxs, sjxs, qtxs
FROM EDU_COURSE_PRACTICE
WHERE kc_flag = '0' AND SFKYKC = '1'
ORDER BY user_kcdm;

-- 查询所有实践环节
SELECT user_kcdm, zwmc, hjlb, zs, sfxs, syxs, sjxs
FROM EDU_COURSE_PRACTICE
WHERE kc_flag = '1' AND SFKYKC = '1'
ORDER BY user_kcdm;

-- 查询某承担单位的课程
SELECT user_kcdm, zwmc, kc_flag, xf, cddw_mc
FROM EDU_COURSE_PRACTICE
WHERE cddw_userdm = 'YX001' AND SFKYKC = '1'
ORDER BY kc_flag, user_kcdm;

-- 查询课程负责人信息
SELECT user_kcdm, zwmc, Kcfzr_XM, Kcfzr, cddw_mc
FROM EDU_COURSE_PRACTICE
WHERE SFKYKC = '1'
ORDER BY cddw_mc, user_kcdm;
```

### 数据插入示例

```sql
INSERT INTO EDU_COURSE_PRACTICE (

) VALUES (

    '计算机学院', 'CS001', '0', 4.0,

```

### 数据更新示例

```sql
-- 更新课程可用状态
UPDATE EDU_COURSE_PRACTICE
SET SFKYKC = '0'
WHERE user_kdcm = 'CS101';

-- 更新课程负责人
UPDATE EDU_COURSE_PRACTICE
SET Kcfzr = 'T002', Kcfzr_XM = '李老师'
WHERE user_kcdm = 'CS101';
```

## 业务逻辑说明

### 课程管理
1. **课程创建**：新增课程的基本信息
2. **课程审核**：审核课程设置的合理性
3. **课程维护**：定期更新课程信息和状态
4. **课程停用**：停用不再使用的课程

### 理论课程特点
1. **教学方式**：以课堂讲授为主
2. **学时分配**：讲授学时占主要比例
3. **考核方式**：通常以考试为主要考核方式
4. **教材使用**：通常配备指定教材

### 实践环节特点
1. **教学方式**：以动手操作为主
2. **时间安排**：通常集中时间段进行
3 **考核方式**：以实践成果为主要考核依据
4. **设备需求**：需要特定的设备或场地

## 数据维护建议

### 定期维护
1. **信息更新**：及时更新课程信息
2. **状态同步**：确保课程状态与实际使用一致
3. **编码维护**：确保课程编码的唯一性和规范性

### 异常处理
1. **重复课程**：检查并处理重复的课程定义
2. **学时异常**：纠正学时分配不合理的情况
3. **状态异常**：处理课程状态设置错误的情况
