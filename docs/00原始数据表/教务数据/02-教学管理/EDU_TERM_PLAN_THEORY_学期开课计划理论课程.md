# EDU_TERM_PLAN_THEORY 学期开课计划（理论课程）表

## 基本信息

- **表名**：`EDU_TERM_PLAN_THEORY`（原表名：`V_BRW_TermPlan_LL`）
- **中文名称**：学期开课计划（理论课程）
- **用途**：存储理论课程的学期开课计划信息，包括教学安排、学时分配、课程属性等
- **字段数量**：24个
- **数据类别**：教务系统 - 教学管理模块

## 字段详细说明

### 学期信息字段
- **XN**：学年，如2009表示2009-2010学年
- **XQ**：学期
  - 0：第一学期
  - 1：第二学期
  - 2：第三学期

### 专业和年级信息
- **NJ**：年级
- **ZYDM**：专业代码
- **ZYMC**：专业名称

### 课程基本信息
- **KCDM**：课程代码
- **KCMC**：课程名称
- **XF**：学分
- **ZZXS**：周学时

### 学时分配字段
- **ZXS**：总学时
- **JSXS**：讲授学时
- **SJXS**：上机学时
- **LWXS**：理论学时
- **SYXS**：实践学时
- **QTXS**：其他学时

### 课程分类字段
- **KCLB1**：课程类别1
- **KCLB2**：课程类别2
- **KCLB3**：课程类别三
- **KCLB4**：课程类别四
- **SJLB**：实践类别

### 教学安排字段
- **CDDWH**：承担单位号
- **CDDWMC**：承担单位名称
- **KHFS**：考核方式
- **XDFS**：修读方式

### 课程性质字段
- **SFZX**：是否主修（1是，0否）
- **SFGX**：是否辅修（1是，0否）

### 其他字段
- **BZ**：备注信息

## 学时分配说明

### 学时构成公式
```
总学时 = 讲授学时 + 上机学时 + 理论学时 + 实践学时 + 其他学时
```

### 理论课程学时特点
- **讲授学时**：占总学时的主要部分
- **理论学时**：纯理论教学学时
- **实践学时**：课程包含的实践环节学时
- **上机学时**：计算机操作类学时

## 课程类别说明

### 课程类别层次结构
- **课程类别1（KCLB1）**：一级分类，如必修/选修
- **课程类别2（KCLB2）**：二级分类，如公共课/专业课
- **课程类别三（KCLB3）**：三级分类，具体课程类型
- **课程类别四（KCLB4）**：四级分类，详细课程属性

### 常见课程类别
- **公共基础课**：全校性基础课程
- **学科基础课**：专业基础理论课程
- **专业核心课**：专业核心理论课程
- **专业选修课**：专业方向选修课程

## 考核方式说明

### 常见考核方式
- **考试**：闭卷或开卷考试
- **考查**：平时成绩综合评定
- **实操**：实际操作考核
- **论文**：课程论文或报告

## 修读方式说明

### 常见修读方式
- **必修**：专业必选课程
- **选修**：学生自主选择课程
- **限选**：指定范围内选择
- **任选**：无限制选择

## 数据约束

1. **非空约束**：XN、XQ、ZYDM、KCDM 字段必须非空
2. **检查约束**：XQ 只能是 0、1、2
3. **检查约束**：SFZX、SFGX 只能是 0 或 1
4. **数值约束**：所有学时字段和学分字段应为非负数
5. **逻辑约束**：总学时应大于等于各分项学时之和

## 索引建议

- **复合主键索引**：`XN, XQ, ZYDM, KCDM` - 确保开课计划唯一性
- **学年索引**：`XN` - 用于按学年查询
- **学期索引**：`XQ` - 用于按学期筛选
- **专业索引**：`ZYDM` - 用于按专业查询开课计划
- **课程索引**：`KCDM` - 用于按课程查询开课安排
- **单位索引**：`CDDWH` - 用于按承担单位查询

## 数据关联关系

### 主要外键关联
- **专业信息表**：通过 `ZYDM` 关联专业基本信息
- **课程信息表**：通过 `KCDM` 关联课程详细信息
- **院系信息表**：通过 `CDDWH` 关联承担单位信息
- **教师信息表**：通过课程安排关联授课教师

### 业务关联
- **课程安排表**：根据开课计划安排具体授课
- **学生选课表**：学生根据开课计划进行选课
- **教室安排表**：根据开课计划安排教室
- **考试安排表**：根据开课计划安排考试

## 使用示例

### SQL查询示例

```sql
-- 查询某学年学期的开课计划
SELECT XN, XQ, ZYMC, KCMC, XF, ZXS, JSXS, KHFS, XDFS
FROM EDU_TERM_PLAN_THEORY
WHERE XN = '2024' AND XQ = '0'
ORDER BY ZYDM, KCDM;

-- 查询某专业的所有开课计划
SELECT XN, XQ, KCMC, XF, ZXS, CDDWMC, KHFS
FROM EDU_TERM_PLAN_THEORY
WHERE ZYDM = 'CS001'
ORDER BY XN, XQ, KCDM;

-- 查询主修课程
SELECT XN, XQ, ZYMC, KCMC, XF, ZXS, KHFS
FROM EDU_TERM_PLAN_THEORY
WHERE SFZX = 1 AND XN = '2024'
ORDER BY XQ, ZYMC, KCMC;

-- 统计学期开课情况
SELECT
    XN,
    XQ,
    COUNT(*) as course_count,
    SUM(XF) as total_credits,
    SUM(ZXS) as total_hours
FROM EDU_TERM_PLAN_THEORY
WHERE XN = '2024'
GROUP BY XN, XQ
ORDER BY XQ;
```

## 业务逻辑说明

### 开课计划制定
1. **专业规划**：根据专业培养计划确定课程设置
2. **师资配置**：根据师资情况安排授课教师
3. **资源分配**：合理安排教室、实验室等教学资源
4. **时间安排**：避免课程时间冲突，优化教学安排

### 学时分配原则
1. **理论为主**：理论课程以讲授学时为主
2. **实践结合**：适当安排实践学时增强理解
3. **学分对应**：学时应与学分合理对应
4. **周学时均衡**：避免周学时过重或过轻

### 课程属性设置
1. **类别准确**：正确设置课程类别便于统计分析
2. **考核合理**：根据课程特点选择合适考核方式
3. **修读明确**：明确主修辅修属性指导学生选课

## 数据维护建议

### 定期维护
1. **计划更新**：及时更新开课计划信息
2. **学时核对**：确保学时分配合理准确
3. **属性维护**：维护课程类别和其他属性信息
4. **单位同步**：确保承担单位信息准确

### 异常处理
1. **重复计划**：检查并处理重复的开课计划
2. **学时异常**：纠正不合理的学时分配
3. **属性错误**：修正错误的课程属性设置
4. **冲突检测**：检测和处理课程安排冲突
