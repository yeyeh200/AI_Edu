# EDU_TEACHER 教师信息表

## 基本信息

- **表名**：`EDU_TEACHER`（根据相关表字段推导）
- **中文名称**：教师信息
- **用途**：存储教师基本信息，包括工号、姓名、所属院系、职称、联系方式等详细信息
- **字段数量**：约15个（根据其他表关联信息推导）
- **数据类别**：教务系统 - 组织架构模块

## 字段详细说明

### 主键字段
- **gh**：教师工号，教师的唯一标识符

### 基本信息字段
- **xm**：教师姓名，教师的真实姓名
- **sfzh**：身份证号，教师身份证号码

### 院系归属字段
- **yxdm**：院系代码，教师所属院系的代码
- **yxmc**：院系名称，教师所属院系的名称

### 资格信息字段
- **zc**：职称，教师的专业技术职称
- **xl**：学历，教师的最终学历
- **xw**：学位，教师获得的最高学位

### 联系方式字段
- **sjhm**：手机号码，教师联系电话
- **email**：电子邮箱，教师邮箱地址

### 状态和日期字段
- **zt**：状态，教师的在职状态
- **rzrq**：入职日期，教师入职时间
- **zcjsrq**：职称晋升日期，最近一次职称晋升时间

### 财务信息字段
- **jbgz**：基本工资，教师的基本工资

### 其他字段
- **bz**：备注，其他需要说明的信息

## 教师工号说明

### 工号构成规则
- **前缀标识**：通常包含院系或类别标识
- **年份信息**：可能包含入职年份
- **序列编号**：个人的唯一序号
- **示例格式**：CS200401表示计算机学院2004年第1号

### 工号特点
- **唯一性**：全校范围内工号唯一
- **稳定性**：工号应保持相对稳定
- **可读性**：包含有用信息便于识别
- **规范性**：遵循统一的编码规范

1. **主键约束**：`gh` 字段必须唯一且非空

3. **检查约束**：`zt` 应在有效状态范围内
4. **格式约束**：`email` 应符合邮箱格式
5. **外键约束**：`yxdm` 应引用院系表的有效记录

## 使用示例

### SQL查询示例

```sql
-- 查询某院系所有教师
SELECT gh, xm, zc, xl, xw, sjhm, email
FROM EDU_TEACHER
WHERE yxdm = 'CS001' AND zt = '1'
ORDER BY zc, xm;

-- 查询具有教授职称的教师
SELECT gh, xm, yxmc, zcjsrq, rzrq
FROM EDU_TEACHER
WHERE zc = '教授' AND zt = '1'
ORDER BY zcjsrq;

-- 按职称统计教师数量
SELECT
    zc,
    COUNT(*) as teacher_count,
    COUNT(DISTINCT yxdm) as dept_count
FROM EDU_TEACHER
WHERE zt = '1'
GROUP BY zc
ORDER BY teacher_count DESC;

-- 查询近三年入职的教师
SELECT gh, xm, yxmc, zc, rzrq
FROM EDU_TEACHER
WHERE rzrq >= DATE_SUB(CURRENT_DATE, INTERVAL 3 YEAR)
ORDER BY rzrq DESC;

-- 查询具有博士学位的教师
SELECT gh, xm, yxmc, zc, xw, email
FROM EDU_TEACHER
WHERE xw = '博士学位' AND zt = '1'
ORDER BY yxmc, xm;
```
