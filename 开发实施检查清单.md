# 开发实施检查清单

### 阶段1：任务理解确认
- [ ] **任务目标明确**
  - [ ] 我清楚要完成什么具体任务
  - [ ] 我理解这个任务的业务目标是什么

- [ ] **约束条件明确**
  - [ ] 我了解所有技术约束
  - [ ] 我了解所有业务约束
  - [ ] 我了解所有时间约束

### 阶段2：文档依赖验证
- [ ] **需求规格检查**
  - [ ] 已查阅 `docs/设计文档_MVP/01-需求与范围/01-MVP需求规格说明书.md`
  - [ ] 确认当前任务在MVP范围内
  - [ ] 确认业务目标与需求文档一致
  - [ ] 确认验收标准明确

- [ ] **架构设计检查**
  - [ ] 已查阅 `docs/设计文档_MVP/02-架构设计/01-系统架构设计.md`
  - [ ] 确认技术方案符合架构设计
  - [ ] 确认组件依赖关系清晰
  - [ ] 确认技术栈使用正确

- [ ] **专项设计文档检查**
  - [ ] 如果是API开发，已查阅API接口规范

## 使用说明

1. **每个开发阶段开始前**：使用对应的检查清单进行验证
2. **每个开发步骤完成后**：使用相应的检查项进行确认
3. **每个里程碑达到时**：使用里程碑验证清单进行全面检查
4. **发现问题时**：使用偏离纠正检查清单进行问题处理

**重要提醒：每个检查项都必须认真验证，不要跳过任何步骤！**
